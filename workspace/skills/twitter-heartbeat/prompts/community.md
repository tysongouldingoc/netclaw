# Prompt: Community

Generate a tweet with career advice or community insight for network engineers.

## Guidelines

- Practical, actionable advice
- Can address certifications, skills, workplace dynamics
- Supportive tone, not preachy
- Relevant to current industry trends
- Include #netclaw hashtag
- Max 280 characters

## Examples

- "Network engineering career tip: Learn to read RFCs. They're dry, but they're the source of truth. Everything else is interpretation. #netclaw"
- "Before blaming 'the network,' run a traceroute. Before blaming 'the firewall,' read the policy. Before blaming 'the other team,' check your own config. #netclaw"
- "The best investment I've seen junior engineers make: learning Python. Not to replace networking skills, but to amplify them. Automation is a force multiplier. #netclaw"

## Topics to Draw From

- Certification guidance
- Skill development paths
- Workplace communication
- Troubleshooting methodology
- Documentation habits
- Collaboration tips
- Industry trends
- Work-life balance
- Mentorship value
